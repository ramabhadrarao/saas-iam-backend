API Documentation

Overview

This documentation provides details on all available API endpoints for the Multi-Tenant SaaS Platform. The base URL for all API requests is:

http://localhost:5001/api/v1

All requests except authentication endpoints require a valid JWT token in the Authorization header:

Authorization: Bearer your_jwt_token_here

Authentication Module

Endpoint Method Description Request Body Example
/auth/login POST Authenticate a user and get a JWT token 
{
  "email": "string",
  "password": "string",
  "subdomain": "string" (optional)
}
 
curl -X POST http://localhost:5001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "Admin123!"
  }'
/auth/refresh-token POST Refresh an expired JWT token 
{
  "refreshToken": "string"
}
 
curl -X POST http://localhost:5001/api/v1/auth/refresh-token \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "your_refresh_token_here"
  }'
/auth/logout POST Log out a user None 
curl -X POST http://localhost:5001/api/v1/auth/logout \
  -H "Authorization: Bearer your_jwt_token_here"
/auth/forgot-password POST Request a password reset 
{
  "email": "string",
  "subdomain": "string" (optional)
}
 
curl -X POST http://localhost:5001/api/v1/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com"
  }'
/auth/reset-password/:token POST Reset password using a token 
{
  "password": "string",
  "subdomain": "string" (optional)
}
 
curl -X POST http://localhost:5001/api/v1/auth/reset-password/your_reset_token_here \
  -H "Content-Type: application/json" \
  -d '{
    "password": "NewPassword123!"
  }'

User Management Module

Endpoint Method Description Request Body/Params Example
/users POST Create a new user 
{
  "firstName": "string",
  "lastName": "string",
  "email": "string",
  "password": "string",
  "userType": "string",
  "tenantId": "string" (optional)
}
 
curl -X POST http://localhost:5001/api/v1/users \
  -H "Authorization: Bearer your_jwt_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "John",
    "lastName": "Doe",
    "email": "john@example.com",
    "password": "Password123!",
    "userType": "tenant_user",
    "tenantId": "tenant_id_here"
  }'
/users GET Get all users Query params:
  • page: Page number (default: 1)
  • limit: Items per page (default: 10)
  • userType: Filter by user type
  • tenantId: Filter by tenant ID
 
curl -X GET "http://localhost:5001/api/v1/users?page=1&limit=10&userType=tenant_admin" \
  -H "Authorization: Bearer your_jwt_token_here"
/users/me GET Get current user profile None 
curl -X GET http://localhost:5001/api/v1/users/me \
  -H "Authorization: Bearer your_jwt_token_here"
/users/:id GET Get user by ID Path param: id 
curl -X GET http://localhost:5001/api/v1/users/user_id_here \
  -H "Authorization: Bearer your_jwt_token_here"
/users/:id PUT Update a user 
{
  "firstName": "string",
  "lastName": "string",
  "isActive": boolean
}
 
curl -X PUT http://localhost:5001/api/v1/users/user_id_here \
  -H "Authorization: Bearer your_jwt_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "John",
    "lastName": "Smith",
    "isActive": true
  }'
/users/assign-role POST Assign a role to a user 
{
  "userId": "string",
  "roleId": "string"
}
 
curl -X POST http://localhost:5001/api/v1/users/assign-role \
  -H "Authorization: Bearer your_jwt_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "user_id_here",
    "roleId": "role_id_here"
  }'
/users/remove-role POST Remove a role from a user 
{
  "userId": "string",
  "roleId": "string"
}
 
curl -X POST http://localhost:5001/api/v1/users/remove-role \
  -H "Authorization: Bearer your_jwt_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "user_id_here",
    "roleId": "role_id_here"
  }'

Audit Logs Module

Endpoint Method Description Request Body/Params Example
/audit-logs GET Get audit logs Query params:
  • page: Page number (default: 1)
  • limit: Items per page (default: 15)
  • search: Search term
  • action: Filter by action
  • module: Filter by module
  • startDate: Filter from date
  • endDate: Filter to date
  • userId: Filter by user ID
  • tenantId: Filter by tenant ID
 
curl -X GET "http://localhost:5001/api/v1/audit-logs?page=1&limit=20&module=AUTH&action=LOGIN" \
    -H "Authorization: Bearer your_jwt_token_here"
/audit-logs/export GET Export audit logs as CSV Same query params as /audit-logs 
curl -X GET "http://localhost:5001/api/v1/audit-logs/export?module=AUTH&action=LOGIN" \
    -H "Authorization: Bearer your_jwt_token_here" \
    -o audit-logs.csv
